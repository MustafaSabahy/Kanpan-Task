## âœ¨ Features

- **Kanban Board Management**
  - Three-column layout: To Do, In Progress, Done
  - Drag and drop tasks between columns
  - Horizontal scrolling for columns
  - Vertical scrolling for tasks within columns
  - Smooth animations with `BouncingScrollPhysics`

  - Create, edit, and delete tasks
  - Task titles and descriptions
  - Task search functionality
  - Clear all tasks or clear tasks from specific columns
  - Integration with Todoist API v2

  - Real-time timer for tasks in "In Progress" state
  - Automatic timer start/stop on column transitions
  - Persistent time tracking across app restarts
  - Time history with detailed session logs
  - Immutable time session records
  - Accurate time calculation using system clock

- **Statistics & Analytics**
  - Overall summary (total time, completed tasks, average time, sessions)
  - Time by status visualization (pie/stacked bar charts)
  - Time by task ranking (horizontal bar charts)
  - Time over time trends (line/bar charts)
  - Advanced filtering (date range, task, status)
  - Empty state handling

- **Task History**
  - Chronological list of time sessions per task
  - Session details: start time, end time, duration, status change reason
  - Complete audit trail of task time tracking

- **Local Push Notifications**
  - Schedule notifications for tasks with due dates
  - Notifications work in background/closed app
  - Update notifications when task time changes
  - Cancel notifications when task is marked Done
  - iOS and Android compatible

- **Dark Mode**
  - Full dark mode support throughout the app
  - Smooth theme transitions
  - Theme persistence

- **Splash Screen**
  - Beautiful animated splash screen
  - Gradient backgrounds (light/dark mode)
  - Smooth fade and scale animations
  - Professional branding

## ğŸ“‹ Column State Management & Time Tracking Cases

### To Do Column

**Behavior:**
- Tasks start here by default
- No timer running
- Can be moved to "In Progress" or "Done"

**Time Tracking Cases:**

1. **Initial State (New Task)**
   - Timer: Not started
   - Time accumulated: 0
   - History: No sessions

2. **From In Progress â†’ To Do**
   - Timer: Stops immediately
   - Time accumulated: Saved to total (elapsed time added)
   - History: Closes active session with reason "paused" or "moved"
   - Start time: Cleared
   - Is running: false

3. **From Done â†’ To Do**
   - Timer: Remains stopped
   - Time accumulated: Preserved (unchanged)
   - History: No new session created
   - Previous sessions: Remain intact

4. **Direct Move To Do â†’ Done (Without Starting)**
   - Timer: Never started
   - Time accumulated: Remains 0
   - History: No session created

### In Progress Column

**Behavior:**
- Timer automatically starts when task enters this column
- Only one task can have an active timer at a time
- Timer survives app restarts and widget rebuilds
- Can be moved back to "To Do" or forward to "Done"

**Time Tracking Cases:**

1. **From To Do â†’ In Progress**
   - Timer: Starts immediately
   - Start time: Set to current time
   - Is running: true
   - History: Creates new session with reason "start"
   - Previous time: Preserved (if any)

2. **From Done â†’ In Progress (Reopen)**
   - Timer: Starts immediately
   - Start time: Set to current time
   - Is running: true
   - History: Creates new session with reason "reopened"
   - Previous time: Preserved (not reset)
   - Old sessions: Remain immutable

3. **Already In Progress (No Change)**
   - Timer: Continues running
   - Time: Calculated on-the-fly (totalTrackedTime + elapsed)
   - History: Active session continues
   - No new session created

4. **App Restart While In Progress**
   - Detection: Checks storage for isRunning = true
   - Previous session: Closed with reason "app_killed"
   - New session: Started with reason "resumed"
   - Start time: Updated to current time
   - Time preservation: Previous elapsed time saved

5. **Widget Rebuild While Running**
   - Timer: Continues running
   - Time calculation: Uses stored startTime + current time
   - State: Preserved across rebuilds
   - No session changes

6. **Multiple Rapid Column Changes**
   - Ensures: No overlapping sessions
   - Each transition: Properly closes previous session
   - Time accuracy: Maintained throughout

### Done Column

**Behavior:**
- Timer stops permanently
- Total tracked time is final and preserved
- Can be reopened to "In Progress" (resumes timer)
- Can be moved back to "To Do" (timer remains stopped)

**Time Tracking Cases:**

1. **From In Progress â†’ Done**
   - Timer: Stops immediately
   - Time calculation: Elapsed time added to totalTrackedTime
   - Start time: Cleared
   - Is running: Set to false
   - History: Closes active session with reason "done" or "completed"
   - Final time: Saved permanently

2. **From To Do â†’ Done (Without Starting)**
   - Timer: Never started
   - Time accumulated: Remains 0
   - History: No session created
   - Completed at: Timestamp saved

3. **Already Done (No Change)**
   - Timer: Remains stopped
   - Time: Unchanged
   - History: No modifications
   - Idempotent: Safe to move to Done multiple times

4. **From Done â†’ In Progress (Reopen)**
   - Timer: Starts immediately
   - Start time: Set to current time
   - Is running: true
   - History: Creates new session with reason "reopened"
   - Previous time: Preserved (not reset)
   - Old sessions: Remain immutable

5. **From Done â†’ To Do**
   - Timer: Remains stopped
   - Time accumulated: Preserved (unchanged)
   - History: No new session created
   - Completed at: Cleared (task no longer marked as done)

### Edge Cases & Special Scenarios

1. **App Killed While Timer Running**
   - On restart: Detects isRunning = true from storage
   - Action: Closes previous session with "app_killed" reason
   - New session: Starts if task still in In Progress
   - Time: Previous elapsed time preserved

2. **Rapid State Changes**
   - Protection: Ensures no overlapping sessions
   - Each transition: Properly handled
   - Time accuracy: Maintained

3. **Multiple Tasks in In Progress**
   - Rule: Only one timer active at a time
   - Behavior: Starting new timer stops previous one
   - Previous time: Saved before stopping

4. **Task Deletion**
   - Time tracking: Deleted with task
   - History: All sessions removed
   - Notifications: Cancelled

5. **Clear Column Actions**
   - In Progress cleared: All running timers stopped and saved
   - Done cleared: Time preserved (already saved)
   - To Do cleared: No timer impact

## ğŸ› Known Issues & Solutions

### Large Task IDs

**Issue**: Task IDs larger than 32-bit integers cause errors in notifications.

**Solution**: Use `hashCode` for notification IDs instead of parsing task IDs as integers.

### Completed Tasks Not Showing

**Issue**: Todoist API doesn't return completed tasks by default.

**Solution**: Fetch completed tasks individually using stored time tracking data.

### Timer State on App Restart

**Issue**: Timer state lost on app restart.

**Solution**: Detect running timers from storage and resume automatically.
